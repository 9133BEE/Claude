使用說明

  • 貼到 TradingView → Pine 編輯器 → 儲存 → 加到圖表
  • Auto 模式：切換圖表時框時自動調整樞紐與搜尋參數，無需手動設定
  • 進場邏輯：等待 MSS（結構突破）→ 找到 OB → 價格回測 OB 區間 → ▲/▼ unicorn 出現再進場
  • 出場邏輯：結構反轉時自動出現「⚠ 多單/空單出場」標籤，提示平倉時機
  • +OB / -OB x/4：品質分數，3分以上升級為 HQ（紫色），優先交易高分 OB
  • HTF OB 方框（1H/4H/1D）：大時框訂單塊，可作為目標區間或進場參考
  • HTF FVG 方框（虛線）：大時框公平價值缺口，被填滿後自動消失
  • 止損建議放在 OB 另一端（頂部或底部）外側 1–2 個 tick

誠實揭露(務必閱讀)

BOS/CHOCH、Order Block都是用pivot高低點與K棒實體比例/ATR位移門檻做的簡化判斷,不是機構訂單簿或真正逐筆成交資料,強趨勢或低流動性時段可能延遲或誤判。RSI/OBV背離用固定回看根數比較高低點,不是嚴格pivot對pivot的常規背離。CVD/Delta同樣是近似值(靠K棒收盤位置推算),Pine Script沒有真正的bid/ask footprint資料。多層確認訊號只是把雜訊訊號濾掉,不代表任何保證勝率——真實表現請用下方的Strategy Tester回測版,對你實際要交易的商品與週期分開跑歷史數據。

回測 ATO V1s
› TradingView Strategy Tester 專用,非即時看盤指標;拿掉儀表板/BOS-CHOCH/OB-FVG/RSI-OBV等視覺模組,只保留核心結構+訂單流確認+進出場
› 進場：亞洲盤高低點被掃 + CVD/Delta背離或轉向 + 價格收回區間/前日高低之上下 + 實體比例過濾,三層同時成立才進場
› 出場：跌破/漲破失效位(亞洲盤區間與前日高低取極值)
› 部位大小：固定口數或風險百分比(依每點/每檔價值換算口數)兩種模式可選
› 想知道這套邏輯的真實勝率,自己對NQ/ES/GC/FX分開回測、分開檢查不同年份區間,過去表現不保證未來,務必留意樣本數與是否過擬合
